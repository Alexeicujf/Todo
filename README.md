# To-Do Application

Веб-приложение для управления задачами и списками дел с поддержкой темного режима и переключения языков интернационализации.

## Архитектура и структура проекта

### Фронтенд (Frontend)

```text
src/
├── api/                  # Функции сетевых запросов (instance, auth, list, todos)
├── assets/               # Статические ресурсы (иконки, шрифты, стили)
├── components/           # Переиспользуемые компоненты интерфейса
│   ├── list/             # Логика списков (CreateListTrigger, CreateListModal, ItemList)
│   ├── to-do/            # Логика задач (CreateTodo, ItemTodo, ListTodos, CreateTodoModal)
│   └── ui/               # Базовые UI-элементы (Button, Input, Modal)
├── config/               # Конфигурационные файлы (роутинг)
├── context/              # Контексты React (ThemeContext, LanguageContext)
├── pages/                # Компоненты страниц приложения
├── store/                # Управление глобальным стейтом (Redux Toolkit)
│   ├── hooks/            # Кастомные типизированные хуки Redux
│   ├── middlewares/      # Кастомные прослойки (logger, localStorage)
│   └── slices/           # Редюсеры и асинхронные санки (authSlice, ListsSlice, TodosSlice)
├── App.css               # Глобальные стили приложения (Tailwind v4)
├── App.tsx               # Корневой компонент приложения с провайдерами
└── main.tsx              # Точка входа клиентской части
```

### Стек технологий

- **Frontend**: React 19, TypeScript, Redux Toolkit, React Router 7, Tailwind CSS v4, Axios, Vite.
- **Backend**: Node.js (чистый HTTP-сервер без фреймворков), Better-SQLite3.

## Запуск проекта

### 1. Бэкенд

Перейдите в папку сервера, убедитесь в наличии настроенного файла `.env` и запустите процесс:

```bash
npm run dev
```

Сервер запустится в режиме разработки на порту `5001`.

### 2. Фронтенд

Перейдите в папку клиентской части и запустите локальный сервер разработки:

```bash
npm run dev
```

Приложение откроется в браузере по адресу `http://localhost:5173` (или на следующем свободном порту).

## Переменные окружения (Environment Variables)

Для работы приложения необходимо создать файлы `.env` в корневых директориях фронтенда и бэкенда на основе файлов `.env.example`.

### Фронтенд (.env)

- `VITE_PORT` — Порт, на котором локально запускается клиентское приложение Vite (по умолчанию `5173`).
- `VITE_API_URL` — URL-адрес бэкенд-сервера, к которому фронтенд отправляет API-запросы (по умолчанию `http://127.0.0.1:5001`).
